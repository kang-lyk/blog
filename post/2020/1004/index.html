从Promise到async和await

Published On: 2020-02-28, Reading Time: 3 minutes

Promise 是ES6异步编程的一种解决方案,js传统的回调函数和事件更合理更强大。es7标准引入了async/ await函数使得异步操作更加方便。async函数就是 Generator 函数的语法糖。 Generator 语法行为与传统函数完全不同，不容易理解，本文直接会直播跳过 Generator 讲asyn / await。

Promise的使用方法

Promise 对象是一个构造函数，用来生成 Promise 实例，如：

const promise = new Promise(function(resolve, reject) {
    // ... some code

  if (/* 异步操作成功 */){
		resolve(value)
	} else {
		reject(error)
  }
})

resolve 在异步操作成功时调用,并将异步操作报出的结果作为参数传递出去。

reject 在异步操作失败时调用，并将异步操作报出的错误作为参数传递出去。

Promise 对像的使用方法如下：

promise.then(function(value) {
  // success
}, function(error) {
  // failure
});

把jq的ajax函数传成Promise对象。

const getList = function(option) {
  return new Promise((resolve, reject) => {
      $.ajax({
        url: '/api/news/list',
        data: option,
        success: function(data) {
          resolve(data)
        },
        error: function(error) {
          reject(error)
        }
      })
  })
}

使用方法

// 第一种写法
getList({page: 1, limit: 10})
	.then((data) => {
		// success
	})
	.catch((error) => {
		// failure 
	})

// 第二种写法
getList({page: 1, limit: 10})
	.then((data) => {
		// success
	}, (error) => {
		// failure 
	})

也可以在then中返回一新的Promise串行使用,避免了回调

getList({page: 1, limit: 10})
	.then((data) => {
		return getList({page: 2, limit: 10})
	})
	.then((data) => {
		return getList({page: 3, limit: 10})
	})
	.catch((error) => {
		// failure 
	})

Promise的状态

每个Promise对象代表一个异步操作，有三种状态：pending（进行中）、fulfilled（已成功）和rejected（已失败）。 每个状态只会出来一次。

const promise = new Promise((resolve, reject) => {
  resolve('第一次resolve')
  conole.log(1)
  reject('error')
  resolve('第二次resolve')
})

promise.then((res) => {
	console.log(res)
})
.catch((error) => {
  console.log(error)
})

// 运行结果
1
'第一次resolve'

这段代码里promise初始状态为pending,在函数体里resolve指它重置为fulfilled状态，每个状态只能出现有一次，所以后面的resovle和reject都无法生效。

Promise.resolve()`等价于下面的写法:

Promise.resolve('foo')
// 等价于
new Promise(resolve => resolve('foo'))

Promise值穿透问题：

Promise.resolve(1)
	.then(Promise.resolve(2))
	.then(3)
	.then()
	.then(console.log)

// 运行结果
1

then方法只允许我们传两个参数：onFulfilled(成功状态处理函数)和onRejected(失败状态的处理函数)。可以两者都传，但只能是函数，其他东西then认为是无效的，所以出现值穿透现象。

将代码改造一下，就不会出现穿透现象了。

const promise = function(data){
  return new Promise((resolve, reject) => {
    resolve(data)
	})
} 

promise(1)
  .then(() => {
  	return promise(2)
	})
	.then(() => {
  	return promise(3)
	})
	.then(console.log)

async函数

async与await的用法

const promise = function(num){
  return new Promise((resolve, reject) => {
    if(num > 10) {
      resolve(num)
    } else {
      reject('error')
    }
	})
} 

const fun = async function() {
  const data = await promise(12)
  console.log('await之后')
}

fun()

通上述代码可知 async函数中的代码一旦遇到await就会先返回，等到异步操作完成，再接着执行函数体内打到车面的说语句。

  • await 后面必须是个Promise对象
  • 如果函数前面不加关键字 async那await将无法使用

async函数语法上规则总体比较简单，难点是错误处理机制。下面修改上述的fun函数对介绍几种错误处理机制。

方法一：

const fun = async function() {
  return await promise(12)
}

fun()
	.then(() => {
  	// do some thing
	})
	.catch((error) => {
  	console.log(error)
	})

方法二：

const fun = async function() {
  try {
    await promise(12)
  	await promise(9)
  } catch(e) {
    throw new Error('出错了');
  }
}

fun()

方法三：

const fun = async function() {
	let num0 = await promise(12)
  	.then(data => data)
  	.catch(() => {
    	throw new Error('出错了');
  }) 
	let num1 = await promise(9)
  	.then(data => data)
  	.catch(() => {
    	throw new Error('出错了');
  })
  return {num0, num1}
}

fun()
	.then(({num0, num1}) => {
  	console.log(num0, num1)
	})
	.catch((error) => {
  	console.log(error)
	})

上面几种方法都不灵活，下面推荐终极解决文案 await to

原理很简单，就是返回两个值

const fun = async function() {
	let [error, data] = await promise(12)
  	.then(num => [null, num])
  	.catch((error) => [error, null]) 
}

数组第一个元素是错误信息，第二个是需要的数据。这也类似go语言的错误处理方式。

这个方法也被人封装成await-to.js

npm i await-to-js安装之后

import to from './to.js';

async function asyncTask() {
     let err, user, savedTask;

     [err, user] = await to(UserModel.findById(1));
     if(!user) throw new CustomerError('No user found');

     [err, savedTask] = await to(TaskModel({userId: user.id, name: 'Demo Task'}));
     if(err) throw new CustomError('Error occurred while saving task');

    if(user.notificationsEnabled) {
       const [err] = await to(NotificationService.sendNotification(user.id, 'Task Created'));  
       if (err) console.error('Just log the error and continue flow');
    }
}

与其他处理方式相比明显更优雅，await-to.js源码很少，不想安装自己写一个也是可以的。
